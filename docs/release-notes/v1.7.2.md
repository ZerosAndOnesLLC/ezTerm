# ezTerm v1.7.2

Patch on top of v1.7.1: local shells now load your shell environment, and the SFTP pane handles multi-file transfers in both directions.

## What changed

### Local shells load your environment — fix (#104)

- **`src-tauri/src/local/mod.rs`** — local Unix shells were spawned as a login shell (`bash -l`). A login shell reads `/etc/profile` and `~/.profile` / `~/.bash_profile` but **skips** `~/.bashrc`, which is where most Linux users keep `PATH` and `export`s. The result: env vars that worked over SSH appeared not to load in a local-shell tab.
- Local shells now spawn interactive (`bash -i` / `zsh -i` / `fish -i`), matching gnome-terminal / Konsole / iTerm. `~/.bashrc` (and the equivalent rc file) loads on every new tab, so your `PATH`, aliases, and exports are present.
- Note: a missing MOTD banner is expected, not a bug — `pam_motd` only fires on a real PAM login (sshd, `login`), never on a terminal-emulator-spawned shell.

### SFTP multi-file download — fix

- **`ui/components/sftp-pane.tsx`** — right-clicking a file and choosing Download only ever downloaded the right-clicked entry, ignoring any active multi-selection.
- When the clicked file is part of a multi-selection, the context menu now reads **"Download N files…"**, prompts **once** for a destination folder, and downloads every selected file into it. Each file gets its own row in the transfer-progress strip. Directories in the selection are skipped (the download path is file-only). Single-file right-click is unchanged.

### SFTP multi-file upload — fix

- **`ui/components/sftp-pane.tsx`** — the toolbar Upload button's file picker was single-select. It now allows picking multiple files; each is uploaded and tracked as its own transfer, and a per-file failure no longer aborts the rest of the batch. Drag-and-drop into the pane already supported many files (and recursive directories) — unchanged.
