# ezTerm v1.7.0

Minor release: the file-browser pane now works for WSL tabs, matching MobaXterm's behaviour. SSH tabs continue to use a real SFTP subsystem; WSL tabs use the Plan 9 bridge Windows already exposes at `\\wsl.localhost\<distro>\…` and shell out to the distro's coreutils for POSIX-aware listing and `chmod`. Plain local shells (`cmd`, `pwsh`) deliberately remain pane-less — same as MobaXterm.

## What changed

### File pane for WSL — new

- **`src-tauri/src/wslfs/`** (new module) — WSL filesystem adapter that mirrors the SSH SFTP surface so the existing `sftp_*` Tauri commands and `SftpPane` UI dispatch transparently for both kinds.
  - **`handle.rs`** — `WslFsHandle { distro, user, home }`. Linux → UNC path translation (`/foo/bar` → `\\wsl.localhost\<distro>\foo\bar`) and the `wsl.exe -d <distro> [-u <user>] -e env LANG=C LC_ALL=C …` runner used for POSIX-aware ops. `$HOME` is resolved once on first realpath and cached for the lifetime of the connection.
  - **`list.rs`** — Directory listing via `ls -la --time-style=+%s`, parsed token-by-token to preserve filenames containing internal spaces. Symbolic mode string (`drwxr-xr-x`, `lrwxrwxrwx`, …) is translated back into the POSIX mode word, including the high-nibble file-type bits, so the frontend's Mode column matches between SSH and WSL panes.
  - **`ops.rs`** — `mkdir` / `rmdir` / `remove` / `rename` go through the UNC path via `tokio::fs` for speed; `chmod` and `realpath` shell out so setuid/setgid/sticky bits survive (the Plan 9 bridge maps perms to Windows ACLs and can't represent them).
  - **`transfer.rs`** — Chunked upload / download mirroring `sftp/transfer.rs` byte-for-byte on the event side (same `sftp:transfer:{id}` channel + `TransferProgress` payload), so the `TransferStatus` strip in the pane needs no kind-aware branch.
- **`src-tauri/src/sftp/registry.rs`** — `SftpRegistry` now stores `FileBrowser { Sftp(SftpHandle) | Wsl(WslFsHandle) }`. One registry, one set of commands, dispatch on the enum.
- **`src-tauri/src/sftp/upload_stream.rs`** — Streaming-upload writer now dispatches by browser kind (`sftp_writer` against russh-sftp; `wsl_writer` against a `tokio::fs::File` on the UNC path). Same begin/chunk/finish/abort lifecycle either way.
- **`src-tauri/src/commands/sftp.rs`** — All `sftp_*` commands now match on `FileBrowser`. `sftp_open` resolves SSH connections from `state.ssh` first, falls back to `state.local` for WSL tabs (rejecting plain local shells with a clear message).
- **`src-tauri/src/local/{mod,registry}.rs`** — `local::Connection` carries an optional `WslMeta { distro, user }`, populated at spawn time when the session kind is `wsl`. That's what `sftp_open` reads to construct a `WslFsHandle`.
- **`src-tauri/src/commands/local.rs`** — `local_disconnect` now drops the file-browser registry entry before tearing down the PTY, keeping cleanup symmetric with the SSH path.
- **`ui/components/tab-slot.tsx`**, **`ui/components/tabs-shell.tsx`** — File-pane toggle + mount now lights up for both `ssh` and `wsl` session kinds. Tooltip text generalised from "SFTP pane" to "file pane".

### Notes on coverage

- **WSL2 with GNU coreutils (Ubuntu, Debian, Kali, openSUSE, Oracle, etc.):** full feature parity with the SSH pane — list, mkdir, rmdir, remove, rename, chmod, upload (drag-drop + file dialog + streaming), download.
- **Alpine / busybox-only distros:** `ls -la --time-style=+%s` doesn't exist in busybox, so the listing call fails with a clear error. `apk add coreutils` fixes it; first-class busybox support is a follow-up.
- **WSL1:** untested (the UNC bridge is WSL2-era). May or may not work depending on host Windows build.

### Terminal — fixes #103 (window shrink not re-fitting)

- **`ui/components/terminal.tsx`** — The xterm resize wiring used to live inside `runConnect`, so the `ResizeObserver` only existed after a successful auth handshake. Any resize during the handshake was dropped, and on WebView2 the observer doesn't always fire on a window-shrink that comes from a manual drag (it fires reliably on growth but not on certain shrink paths). When that happened, xterm kept its old `cols`/`rows`, Claude / vim / less never received SIGWINCH, and the rendered text extended past the visible viewport.
- Resize wiring now lives in the mount-time effect alongside `terminal.open()` so it's live the instant xterm renders, regardless of connect state. Both `ResizeObserver` *and* `window.addEventListener('resize', …)` feed the same rAF-coalesced handler — the window event covers the WebView2 shrink case where the observer doesn't fire, the observer covers in-app layout shifts (SFTP pane toggle, view-mode switch) where there's no window event, and the rAF gate still ensures one IPC per animation frame.

### Version alignment

- **`Cargo.toml`** (workspace), **`src-tauri/tauri.conf.json`**, **`ui/package.json`** all now report `1.7.0` so the installed app and the auto-updater agree on what's running.
