# ezTerm v1.3.8

Patch: right-click → Paste now actually focuses ezTerm when the app is in the background.

## What changed

- **`ui/components/terminal.tsx`** — Both the terminal's `contextmenu` handler and `doPaste` now call `getCurrentWindow().setFocus()`, and `doPaste` also re-focuses the xterm instance after writing the pasted bytes. Previously, copying from another app, right-clicking ezTerm, and choosing **Paste** would paste the text but leave the window/terminal unfocused — the user had to left-click in the terminal before they could keep typing. Now the window comes forward on right-click, and after Paste the cursor is live for immediate input.

The Shift+Insert and Ctrl+Shift+V paste paths were unaffected (they only fire when ezTerm is already focused) and continue to work as before.
