# ezTerm v1.11.1

Port forwarding is now a first-class, reliable feature — create, edit, and manage Local (`-L`), Remote (`-R`), and Dynamic / SOCKS5 (`-D`) tunnels per session, with clear live state and no surprising errors. This release folds in the forward-persistence groundwork from the never-published v1.9.0 draft, adds a full reliability pass, and refreshes dependencies to clear outstanding security advisories.

## Port forwarding

- **Editable everywhere** — edit a saved forward from the session editor *or* the per-tab **Forwards** pane, and right-click any forward row for an Edit / Start-Stop / Delete menu. (Previously the pane's Edit button silently saved-and-closed the session dialog, and saved forwards couldn't be edited from the pane at all.)
- **Reliable auto-start** — forwards set to auto-start on connect now show as **running** immediately instead of appearing idle, so you no longer hit *"port already bound"* clicking Start on a tunnel that's already up.
- **Safe edit-in-place** — editing a running forward waits for the old listener to release its bind before re-binding, so same-port edits no longer race into *"address in use."*
- **Graceful conflicts across tabs & windows** — opening the same session in a second tab, or running a second ezTerm, no longer throws a red error when a port is already forwarded. The row shows a neutral amber **"in use elsewhere"** state with Start still available to take it over.
- **Clearer pane** — green **Start** / red **Stop**, live per-forward status dots, and error/conflict messages that stay put instead of vanishing.
- Forwards created from a live tab **persist to the session** (survive reconnects) and auto-restore on the next connect.

## Fixes

- Right-clicking empty areas no longer opens the webview's native menu, whose **Reload wiped every open tab and connection** — the native context menu is suppressed app-wide, leaving only ezTerm's own menus.

## Dependencies & security

- Cleared all outstanding high- and moderate-severity advisories: docs site (astro, vite, undici, dompurify, js-yaml) and the app UI (Next's bundled postcss). Rust `cmov` updated for its RUSTSEC fix.
