# ezTerm v1.12.0

The in-app updater can now track a **pre-release channel**, so you can opt in to test a build before it's promoted to the public release.

## Pre-release updates

- **"Include pre-releases (beta)" toggle** in the updater dialog. When enabled, ezTerm tracks the newest release — pre-release *or* stable, whichever is newer — instead of stable-only.
- Toggling the channel re-checks immediately, and the choice persists so scheduled auto-checks follow the same channel.
- Available pre-release builds are clearly badged **Pre-release** in the dialog, distinct from a normal update.
- The offered pre-release must carry a higher version than what's installed (e.g. `v1.12.0-rc.1` while on `1.11.2`), following standard updater version comparison.
